1. Difference among map, filter, foreach and find?

  • forEach( ), is used to execute the same code on every item in an array but does not change the array and it does not return anything.
  • map( ), is almost same as forEach( ) but map by default returns a list of array and If there are nothing to return then map returns an empty array
  • filter( ), is used to pick all the matching elements which fulfills the condition given.
  • find( ), is almost same with filter( ) but it only returns or picks out the first matching element.

    • 2. Difference among local storage, session storage and cookies?

    Local storage, session storage, and cookies are all mechanisms available in web browsers to store data on the client-side, meaning on the user's computer or device, rather than on the server

  • Local storage: This is a mechanism that allows web apps to store data in the user's web browser. The data stored in local storage stays stored even after the browser is closed and reopened.
  • Session storage: This is similar to local storage, but the data stored in session storage is only accessible in the same browser tab or window, and it is cleared when the tab or window is closed.
  • Cookies are small pieces of text sent to your browser by a website you visit. They help that website remember information about your visit, which can both make it easier to visit the site again and make the site more useful to you.Unlike local storage and session storage, cookies are sent to the server with each request.

    • 3. what is event loop and how does it work?

  • The event loop is a process that waits for the Call Stack to be clear before pushing callbacks from the Task Queue to the Call Stack. Once the Stack is clear, the event loop triggers and checks the Task Queue for available callbacks. If there are any, it pushes it to the Call Stack, waits for the Call Stack to be clear again, and repeats the same process.
